# 宜春学院计算机专业毕业论文格式规范

## 一、论文基本结构

### 1. 前置部分

```
标题（基于 SSM/SpringMVC 的 XXX 系统的设计与实现）
学院信息（宜春学院 数学与计算机科学学院 计算机科学与技术专业 作者姓名）
指导老师信息（指导老师：XXX（校内）XXX（企业））
摘要（中文）
关键字（3-5 个，分号分隔）
Abstract（英文摘要）
Keywords（英文关键字）
目录
```

### 2. 正文部分

#### 第 1 章 绪论（一级标题）
- 1.1 研究背景（二级标题）
- 1.2 研究目的与意义（二级标题）
  - 研究目的（正文样式）
  - 研究意义（正文样式）
- 1.3 研究现状（二级标题）

#### 第 2 章 需求分析（一级标题）
- 2.1 业务分析（二级标题）
  - 2.1.1 业务描述（三级标题）
  - 2.1.2 业务人员分析（三级标题）
- 2.2 系统业务流程分析（二级标题）
  - 各子流程（三级标题）：登录流程、信息修改流程、管理流程等
- 2.3 系统用例分析（二级标题）
  - 2.3.1 用户角色分析（三级标题）
  - 2.3.2 总体用例分析（三级标题）
  - 各用例描述（三级标题）

#### 第 3 章 软件设计（一级标题）
- 3.1 软件体系结构设计（二级标题）
- 3.2 软件功能模块设计（二级标题）
  - 3.2.1 系统总功能设计（三级标题）
  - 3.2.2 用户登录注册模块设计（三级标题）
  - 3.2.3 各功能模块设计（三级标题）
- 3.3 数据库设计（二级标题）
  - 3.3.1 概念模型设计（三级标题）
  - 3.3.2 实体关系图（三级标题）
  - 3.3.3 逻辑模型设计（三级标题）
  - 3.3.4 物理模型设计（三级标题）

#### 第 4 章 系统实现（一级标题）
- 4.1 登录注册实现（二级标题）
- 4.2 主要功能模块实现（二级标题）
- 4.3 其他功能实现（二级标题）

#### 第 5 章 系统测试（一级标题）
- 5.1 测试环境与方法（二级标题）
- 5.2 功能测试（二级标题）
- 5.3 测试结果分析（二级标题）

#### 第 6 章 总结与展望（一级标题）


## 二、样式规范

### 1. 标题样式

| 级别 | Word 样式 | Markdown 等效 | 说明 |
|------|----------|--------------|------|
| 一级标题 | Heading 1 | `#` | 章标题，如"1 绪论"、"2 需求分析" |
| 二级标题 | Heading 2 | `##` | 节标题，如"1.1 研究背景" |
| 三级标题 | Heading 3 | `###` | 小节标题，如"2.1.1 业务描述" |

### 2. 正文样式

| 元素 | 样式名称 | 格式要求 |
|------|----------|----------|
| 普通段落 | 论文正文 | 首行缩进 2 字符 |
| 摘要/关键字 | Normal | 无缩进 |
| 英文摘要 | Normal | 无缩进 |
| 图注 | Normal/论文正文 | 居中，如"图 2-1 XXX 流程图" |
| 表注 | Caption/Normal | 如"表 2-1 XXX 表" |

### 3. 图表规范

**图片格式：**
- 图注位置：图片下方
- 命名格式：`图 X-Y [图片内容描述]`
- 示例：`图 2-1 家属和老人的登录注册用例图`

**表格格式：**
- 表注位置：表格上方
- 命名格式：`表 X-Y [表格内容描述]`
- 示例：`表 2-1 系统角色划分表`
- 续表格式：`(续表 X-Y)`

## 三、写作内容规范

### 1. 摘要写作

**中文摘要结构：**
1. 背景句：阐述研究背景和问题的重要性
2. 目的句：说明系统设计的目标
3. 方法句：描述采用的技术栈和方法
4. 结果句：概述系统实现的功能
5. 结论句：总结系统的价值和意义

**字数要求：** 约 300-500 字

**示例：**
```
摘要：随着老年人口比例持续攀升，社区养老服务体系的建设与完善已成为社会各界高度重视的民生议题。
为了提高社区老人管理效率，提升家属对老人生活的关注度与参与度，保障老人的健康与安全，
本文设计并实现了一个基于 Spring Boot 的社区老人管理系统。
本系统包含家属、老人和管理员三类用户角色，具备多样化的功能...
本系统采用 Spring Boot 框架进行开发，具有良好的可扩展性与维护性，
能够满足社区老人管理的实际需求，为社区老人管理提供了一种高效、便捷的解决方案。
```

**关键字格式：**
- 3-5 个关键字
- 分号分隔
- 示例：`关键字：社区老人管理；Spring Boot；健康；安全`

### 2. 绪论章节

#### 1.1 研究背景
- 描述社会背景和现实意义
- 引用相关数据支撑（使用上标引用 [1][2] 等）
- 分析现有问题
- 提出技术解决方案的必要性

#### 1.2 研究目的与意义
- **研究目的**：具体说明系统要解决的问题和实现的目标
- **研究意义**：
  - 理论层面：学术贡献、技术创新
  - 实践层面：应用价值、社会效益

#### 1.3 研究现状
- 国内研究现状
- 国外研究现状
- 现有研究的不足
- 本研究的创新点

### 3. 需求分析章节

#### 业务分析
- 业务描述：系统定位、服务对象、核心功能
- 业务人员分析：各角色的职责和功能权限

#### 业务流程分析
- 使用流程图说明各业务流程
- 图注格式：`图 2-X XXX 流程图`

#### 用例分析
- 用户角色分析表
- 总体用例图
- 各用例详细描述（用例描述表）

### 4. 软件设计章节

#### 体系结构设计
- 技术栈说明（Spring MVC、MyBatis-Plus、JSP、MySQL 等）
- 分层架构说明（Controller、Service、Mapper、Entity）
- 框架优势分析

#### 功能模块设计
- 总体功能包图
- 各模块缩略类图
- 模块间关系说明

#### 数据库设计
- 数据库选型说明（MySQL 特点）
- 概念模型设计（各实体属性图）
- 实体关系图（E-R 图）
- 逻辑模型设计（关系模式）
- 物理模型设计（数据表结构）

## 五、常见样式错误

1. **标题级别混乱**：确保一级、二级、三级标题层次分明
2. **图表编号不连续**：使用 Word 的题注功能自动生成
3. **参考文献格式不统一**：严格按照国标格式
4. **段落格式不一致**：正文首行缩进 2 字符
5. **英文摘要语法错误**：注意时态和语态

## 六、技术栈描述模板

### 项目实际技术栈

根据超市管理系统项目，采用的技术栈如下：

| 技术 | 版本 | 说明 |
|------|------|------|
| Spring | 5.0.0.RELEASE | 核心框架，提供 IoC 和 AOP 支持 |
| Spring MVC | 5.0.0.RELEASE | Web 层框架，处理请求分发和视图渲染 |
| MyBatis-Plus | 2.3 | 持久层框架，简化数据库操作 |
| MySQL | 5.1.38 | 关系型数据库 |
| Druid | 1.1.0 | 数据库连接池 |
| FastJson | 1.2.8 | JSON 序列化和反序列化 |
| JSP | 2.1 | 前端页面技术 |
| JSTL | 1.2 | JSP 标准标签库 |
| Tomcat | 9.0.29 | Web 应用服务器 |
| Log4j | 1.2.17 | 日志框架 |

### 架构说明

**分层架构：**
- **表示层 (Controller)**: Spring MVC 控制器，负责请求接收和响应返回
- **业务逻辑层 (Service)**: 处理核心业务逻辑
- **数据访问层 (Mapper/DAO)**: MyBatis-Plus 接口，负责数据库操作
- **领域模型层 (Entity)**: JavaBean，封装数据实体
- **前端页面 (JSP)**: 使用 JSP+JSTL 进行页面渲染

### Spring MVC 描述
```
Spring MVC 作为基于 Java 的 Web 层框架，采用模型 - 视图 - 控制器 (MVC) 设计模式，
通过前端控制器 (DispatcherServlet) 统一处理请求，具有配置简单、开发效率高、
与 Spring 框架无缝集成等优点，适合用于构建 B/S 架构的管理系统。
```

### MyBatis-Plus 描述
```
MyBatis-Plus 是在 MyBatis 基础上的增强工具，提供了通用 Mapper、代码生成器、
分页插件等功能，能够简化数据库 CRUD 操作，提高开发效率。它保留了 MyBatis
的灵活性，同时减少了大量样板代码的编写。
```

### MySQL 描述
```
MySQL 是一种开源的关系型数据库管理系统，具有性能卓越、成本低、易用性好等特点。
它支持 SQL 标准，提供事务支持、多表关联查询、存储过程等功能，能够满足中小型
应用系统的数据存储和管理需求。
```

### Druid 描述
```
Druid 是阿里巴巴开源的数据库连接池，具有高性能、高扩展性、高可靠性等特点。
它提供了强大的监控功能，可以实时监控数据库连接池的运行状态，是 Java 应用
中广泛使用的数据库连接池组件。
```

### JSP+JSTL 描述
```
JSP (JavaServer Pages) 是一种基于 Java 的 Web 页面技术，允许在 HTML 中嵌入
Java 代码。JSTL (JSP Standard Tag Library) 是 JSP 标准标签库，提供了一系列
通用标签，可以简化 JSP 页面的开发，减少脚本代码的使用。
```
